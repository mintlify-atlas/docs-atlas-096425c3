When using React Google Maps with Server-side Rendering (SSR) or fullstack frameworks, you must exclude the map from server-side rendering since it's not supported by the Google Maps API.

<Warning>
Google Maps components must be rendered client-side only. The Google Maps JavaScript API does not support server-side rendering.
</Warning>

<Info>
A solution that provides basic SSR capabilities via the Static Maps API is currently being evaluated.
</Info>

## Next.js

Next.js (App Router) requires the `'use client'` directive to mark components as client-only.

### Basic Implementation

<Steps>

<Step title="Create a client component">
Add the `'use client'` directive at the top of your map component file:

```tsx
'use client';

import {APIProvider, Map} from '@vis.gl/react-google-maps';

export default function MyMap() {
  return (
    <div className="container">
      <APIProvider apiKey={process.env.NEXT_PUBLIC_GOOGLE_MAPS_API_KEY!}>
        <Map
          mapId={'bf51a910020fa25a'}
          defaultZoom={5}
          defaultCenter={{lat: 53, lng: 10}}
          gestureHandling={'greedy'}
          disableDefaultUI={true}
        />
      </APIProvider>
    </div>
  );
}
```

<Note>
The `'use client'` directive tells Next.js that this component should only be rendered on the client.
</Note>
</Step>

<Step title="Use environment variables">
Store your API key in `.env.local`:

```bash
NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=your_api_key_here
```

<Warning>
Next.js environment variables must be prefixed with `NEXT_PUBLIC_` to be accessible in client components.
</Warning>
</Step>

<Step title="Import in your page">
Import and use the client component in your page:

```tsx
import MyMap from './components/MyMap';

export default function Home() {
  return (
    <main>
      <h1>My Map Application</h1>
      <MyMap />
    </main>
  );
}
```
</Step>

</Steps>

### Complete Example

Here's a complete Next.js App Router example:

```tsx filename="app/page.tsx"
'use client';

import {APIProvider, Map} from '@vis.gl/react-google-maps';
import styles from './page.module.css';

export default function Home() {
  const API_KEY =
    (process.env.NEXT_PUBLIC_GOOGLE_MAPS_API_KEY as string) ??
    globalThis.GOOGLE_MAPS_API_KEY;

  return (
    <div className={styles.container}>
      <APIProvider apiKey={API_KEY}>
        <Map
          mapId={'bf51a910020fa25a'}
          defaultZoom={5}
          defaultCenter={{lat: 53, lng: 10}}
          gestureHandling={'greedy'}
          disableDefaultUI={true}
        />
      </APIProvider>
    </div>
  );
}
```

### Next.js Pages Router

For the Pages Router, use dynamic imports with `ssr: false`:

```tsx filename="pages/index.tsx"
import dynamic from 'next/dynamic';

const MapComponent = dynamic(
  () => import('../components/Map'),
  { ssr: false }
);

export default function Home() {
  return (
    <div>
      <h1>My Map</h1>
      <MapComponent />
    </div>
  );
}
```

## Remix

Remix applications should use the `<ClientOnly>` component from `remix-utils` to render maps only on the client.

### Installation

<Steps>

<Step title="Install remix-utils">
```bash
npm install remix-utils
```
</Step>

<Step title="Create your map component">
```tsx filename="app/components/map/map.client.tsx"
import {APIProvider, Map} from '@vis.gl/react-google-maps';
import styles from './map.module.css';

export default function MyMap() {
  const API_KEY =
    (process.env.GOOGLE_MAPS_API_KEY as string) ??
    globalThis.GOOGLE_MAPS_API_KEY;

  return (
    <div className={styles.container}>
      <APIProvider apiKey={API_KEY}>
        <Map
          mapId={'bf51a910020fa25a'}
          defaultZoom={5}
          defaultCenter={{lat: 53, lng: 10}}
          gestureHandling={'greedy'}
          disableDefaultUI={true}
        />
      </APIProvider>
    </div>
  );
}
```
</Step>

<Step title="Create a fallback component (optional)">
```tsx filename="app/components/map/map-fallback.tsx"
export default function MapFallback() {
  return (
    <div className="map-fallback">
      <p>Loading map...</p>
    </div>
  );
}
```

<Warning>
If you use a fallback with known dimensions, make sure it has the same size as your final map to prevent layout shifts.
</Warning>
</Step>

<Step title="Use ClientOnly in your route">
```tsx filename="app/routes/_index.tsx"
import {ClientOnly} from 'remix-utils/client-only';
import MyMap from '../components/map/map.client';
import MyMapFallback from '../components/map/map-fallback';

export default function Index() {
  return (
    <ClientOnly fallback={<MyMapFallback />}>
      {() => <MyMap />}
    </ClientOnly>
  );
}
```
</Step>

</Steps>

### Environment Variables in Remix

For Remix, create a `.env` file:

```bash
GOOGLE_MAPS_API_KEY=your_api_key_here
```

Access it in your loader or client code:

```tsx
// In a loader (server-side)
export async function loader() {
  return json({
    apiKey: process.env.GOOGLE_MAPS_API_KEY
  });
}

// In your component
export default function MyMap() {
  const {apiKey} = useLoaderData<typeof loader>();
  
  return (
    <APIProvider apiKey={apiKey}>
      <Map {/* ... */} />
    </APIProvider>
  );
}
```

## Other Frameworks

### Gatsby

For Gatsby, use the `gatsby-plugin-no-javascript` or create a wrapper component that only renders on the client:

```tsx
import React, {useEffect, useState} from 'react';

const MapWrapper = ({children}) => {
  const [isClient, setIsClient] = useState(false);

  useEffect(() => {
    setIsClient(true);
  }, []);

  return isClient ? children : null;
};

export default MapWrapper;
```

### Astro

For Astro, use the `client:only` directive:

```astro
---
import MapComponent from '../components/Map';
---

<MapComponent client:only="react" />
```

## Best Practices

<Tabs>
  <Tab title="Loading States">
    Provide meaningful loading states while the map initializes:

    ```tsx
    function MapFallback() {
      return (
        <div className="map-skeleton" style={{height: '400px'}}>
          <div className="spinner" />
          <p>Loading map...</p>
        </div>
      );
    }
    ```
  </Tab>
  
  <Tab title="Error Handling">
    Handle API loading errors gracefully:

    ```tsx
    import {useApiLoadingStatus} from '@vis.gl/react-google-maps';
    import {APILoadingStatus} from '@vis.gl/react-google-maps';

    function MapWithErrorHandling() {
      const status = useApiLoadingStatus();

      if (status === APILoadingStatus.FAILED) {
        return <div>Failed to load Google Maps</div>;
      }

      return <Map {/* ... */} />;
    }
    ```
  </Tab>
  
  <Tab title="Environment Variables">
    **Security Tips:**
    
    - Never commit API keys to version control
    - Use different keys for development and production
    - Set up API key restrictions in Google Cloud Console
    - Use environment-specific `.env` files

    ```bash
    # .env.local (not committed)
    NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=your_api_key

    # .env.example (committed as template)
    NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=your_api_key_here
    ```
  </Tab>
</Tabs>

## Layout Shift Prevention

<Info>
To prevent cumulative layout shift (CLS), set explicit dimensions for your map container:
</Info>

```css
.map-container {
  width: 100%;
  height: 400px;
  /* or use aspect-ratio for responsive sizing */
  aspect-ratio: 16 / 9;
}
```
